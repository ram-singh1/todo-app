# Deploying to Render (free, no credit card)

This guide takes you from a fresh repo to a live API in about 15 minutes
using only free-tier services that **do not require a credit card**:

| Service       | What it does                       | Free tier             |
|---------------|------------------------------------|-----------------------|
| Render        | Hosts the Node/Express API         | 750 hrs/mo, sleeps after 15 min idle |
| MongoDB Atlas | Hosted MongoDB                     | M0: 512 MB shared cluster |
| Cloudinary    | Image/file storage (Render disk is ephemeral) | 25 GB storage, 25 GB bandwidth/mo |

---

## 1. Push this repo to GitHub

The repo's existing remote points at `todo-app.git`. To push to your new
`todo-bknd` repo:

```bash
cd backend
git remote set-url origin https://github.com/ram-singh1/todo-bknd.git
git add .
git commit -m "Prep for Render deployment"
git push -u origin main
```

If `todo-bknd` already has commits, you may need `git push -u origin main --force`
the first time (only safe if you don't care about its existing history).

---

## 2. Create a MongoDB Atlas cluster (free, no card)

1. Sign up at <https://www.mongodb.com/cloud/atlas/register>.
2. Create a project, then "Build a Database" → choose **M0 Free**.
3. Choose any provider/region; the closest to Render's region is fine.
4. **Database Access** → "Add New Database User" → username + auto-generated
   password. Copy the password somewhere safe.
5. **Network Access** → "Add IP Address" → enter `0.0.0.0/0` (allow from
   anywhere, since Render's outbound IPs aren't fixed on the free tier).
6. Once the cluster is up, click "Connect" → "Drivers" → copy the connection
   string. It looks like:

   ```
   mongodb+srv://<USER>:<PASSWORD>@cluster0.xxxxx.mongodb.net/?retryWrites=true&w=majority
   ```

   Replace `<USER>` and `<PASSWORD>` with the values from step 4. Add a
   database name before the `?`, e.g. `…mongodb.net/todo-diary?retryWrites=…`.
   Save this — you'll paste it into Render in step 4.

---

## 3. Create a Cloudinary account (free, no card)

1. Sign up at <https://cloudinary.com/users/register_free>.
2. After login, on the Dashboard, copy three values:
   - **Cloud Name**
   - **API Key**
   - **API Secret** (click "Reveal")
3. Save these — you'll paste them into Render in step 4.

---

## 4. Deploy on Render (free, no card)

1. Sign up at <https://render.com/register> using your GitHub account
   (this lets Render auto-detect your repos).
2. Click **New** → **Blueprint**.
3. Pick the `todo-bknd` repo. Render reads [`render.yaml`](./render.yaml)
   and offers to create the Web Service.
4. Click **Apply**. The service is created; a few env vars are still empty.
5. Open the new service → **Environment** tab → set the secrets you collected:
   - `MONGODB_URI` → the Atlas connection string from step 2
   - `CLOUDINARY_CLOUD_NAME`
   - `CLOUDINARY_API_KEY`
   - `CLOUDINARY_API_SECRET`
   - `CORS_ORIGINS` → leave blank for now (allows all origins, fine for a
     mobile app where requests are JWT-authenticated)

   `JWT_SECRET` and `ENCRYPTION_KEY` are auto-generated by Render and don't
   need touching.
6. Click **Manual Deploy** → **Deploy latest commit**. First deploy takes
   2-4 minutes; watch the log for `✅ MongoDB Connected` and `✅ Server
   running on port 10000`.
7. Verify by hitting `https://<your-service>.onrender.com/` in a browser —
   you should see a JSON health response.

---

## 5. Point the mobile app at the live API

In the frontend, the API URL now reads from `EXPO_PUBLIC_API_URL`
(see [`frontend/src/api/client.js`](../frontend/src/api/client.js)).

For an Expo EAS build:

```bash
# In frontend/, create a .env file:
echo "EXPO_PUBLIC_API_URL=https://your-service.onrender.com/api" > .env

# Then build:
eas build --profile production --platform android
```

For local dev against the live backend, do the same and restart `expo start`.

## Cold start (the only catch)

Render's free Web Service sleeps after 15 minutes with no traffic. The next
request takes 20-40 seconds while the service wakes back up. After that
it's fast until the next idle window.

If this becomes a problem:
- **Free workaround:** Use a service like <https://cron-job.org> (free, no
  card) to ping `https://<your-service>.onrender.com/` every 14 minutes.
  Keeps the dyno warm 24/7.
- **Paid:** $7/mo Starter plan removes the sleep entirely.

## Troubleshooting

**"Application failed to respond" / 502 on first request**
Cold start in progress. Wait 30s and retry.

**`Missing required env vars: MONGODB_URI` in Render logs**
You skipped step 4.5. Set the env var, redeploy.

**Atlas: "Authentication failed"**
Most often a password copy/paste mistake, or special chars in the password
that need URL-encoding. Either rotate to an alphanumeric password in
Atlas → Database Access, or use `encodeURIComponent` mentally.

**Cron jobs aren't firing while the service sleeps**
Expected on free tier. Either (a) ping the service to keep it warm,
or (b) accept that overdue reminders are caught up on the next wake.

**Uploads return `provider: "local"` instead of Cloudinary URLs in prod**
One of the three Cloudinary env vars is missing. Check Render → Environment.
